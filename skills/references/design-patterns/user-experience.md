# 用户体验设计模式

本文档提供常用的用户体验设计模式和最佳实践。

## 🎨 交互设计模式

### 导航模式

#### 1. 顶部导航栏

```text
┌─────────────────────────────────────────────────┐
│ Logo   [首页] [产品] [服务] [关于]    [搜索] [用户] │
└─────────────────────────────────────────────────┘
```

**适用场景**：

- 网站和 Web 应用
- 导航项较少（5-7 个）
- 需要品牌展示

#### 2. 侧边导航栏

```text
┌──────┬────────────────────────────┐
│ 🏠   │                            │
│ 首页  │                            │
│──────│        内容区域             │
│ 📊   │                            │
│ 数据  │                            │
│──────│                            │
│ ⚙️   │                            │
│ 设置  │                            │
└──────┴────────────────────────────┘
```

**适用场景**：

- 后台管理系统
- 功能模块较多
- 需要深层级导航

#### 3. 底部标签栏

```text
┌─────────────────────────────────┐
│                                 │
│           内容区域               │
│                                 │
├─────────────────────────────────┤
│  🏠     🔍     ➕     💬     👤  │
│  首页   发现   发布   消息   我的 │
└─────────────────────────────────┘
```

**适用场景**：

- 移动端应用
- 主要功能入口
- 3-5 个标签

### 表单设计模式

#### 1. 单列表单

```text
┌─────────────────────────────┐
│ 标题：创建账户               │
├─────────────────────────────┤
│ 用户名 *                    │
│ ┌─────────────────────────┐ │
│ │                         │ │
│ └─────────────────────────┘ │
│                             │
│ 邮箱 *                      │
│ ┌─────────────────────────┐ │
│ │                         │ │
│ └─────────────────────────┘ │
│                             │
│ 密码 *                      │
│ ┌─────────────────────────┐ │
│ │                         │ │
│ └─────────────────────────┘ │
│                             │
│      [取消]    [创建账户]    │
└─────────────────────────────┘
```

**最佳实践**：

- 标签在输入框上方
- 必填项明确标记
- 实时校验反馈
- 主操作按钮突出

#### 2. 分步表单

```text
步骤指示器：
[1. 基本信息] ──── [2. 详细设置] ──── [3. 确认提交]
     ●                  ○                  ○

当前步骤内容...

[上一步]                              [下一步]
```

**适用场景**：

- 复杂表单（字段多于 10 个）
- 需要引导用户完成
- 各步骤相对独立

### 反馈模式

#### 1. Toast 提示

```text
┌───────────────────────┐
│ ✓ 保存成功            │  ← 顶部/底部短暂显示
└───────────────────────┘
```

**适用场景**：操作结果反馈、非阻塞提示

#### 2. 模态对话框

```text
┌─────────────────────────────┐
│ ⚠️ 确认删除                 │
├─────────────────────────────┤
│                             │
│ 此操作不可恢复，确定要删除吗？ │
│                             │
├─────────────────────────────┤
│        [取消]    [删除]      │
└─────────────────────────────┘
```

**适用场景**：需要用户确认的重要操作

#### 3. 内联反馈

```text
邮箱
┌─────────────────────────────┐
│ invalid-email               │
└─────────────────────────────┘
❌ 请输入有效的邮箱地址
```

**适用场景**：表单校验、实时反馈

## 📱 响应式设计

### 断点设计

| 断点名称 | 宽度范围       | 典型设备         |
| -------- | -------------- | ---------------- |
| xs       | < 576px        | 手机竖屏         |
| sm       | 576px - 768px  | 手机横屏、小平板 |
| md       | 768px - 992px  | 平板             |
| lg       | 992px - 1200px | 笔记本           |
| xl       | ≥ 1200px       | 桌面显示器       |

### 响应式策略

#### 1. 内容优先级调整

```text
桌面端：[主内容 70%] [侧边栏 30%]

移动端：
[主内容 100%]
[侧边栏（折叠或隐藏）]
```

#### 2. 导航适配

```text
桌面端：水平导航栏
         [首页] [产品] [服务] [关于]

移动端：汉堡菜单
         [☰]  Logo  [搜索]
```

## 🎭 状态设计

### 组件状态

| 状态     | 视觉表现        | 使用场景 |
| -------- | --------------- | -------- |
| Default  | 正常样式        | 初始状态 |
| Hover    | 背景/边框变化   | 鼠标悬停 |
| Focus    | 明显轮廓        | 键盘聚焦 |
| Active   | 按下效果        | 点击瞬间 |
| Disabled | 灰色/降低透明度 | 不可操作 |
| Loading  | 加载动画        | 等待中   |
| Error    | 红色边框/图标   | 错误状态 |
| Success  | 绿色边框/图标   | 成功状态 |

### 页面状态

```text
│ ░░░░░░░░░░░░░░░░            │
```

#### 2. 空状态

```text
┌─────────────────────────────┐
```

```text

│    [重试]  [返回首页]        │
│                             │
└─────────────────────────────┘
```

## ♿ 无障碍设计

### 基本原则 (WCAG)

1. **可感知**：信息可被用户感知
2. **可操作**：界面可被用户操作
3. **可理解**：信息和操作可被理解
4. **健壮性**：内容可被各种技术解析

### 实践要点

#### 颜色对比度

| 文本类型        | 最小对比度 |
| --------------- | ---------- |
| 正文文本        | 4.5:1      |
| 大文本（18px+） | 3:1        |
| 图形和 UI 组件  | 3:1        |

#### 键盘可访问性

- 所有交互元素可通过 Tab 键访问
- 焦点顺序符合视觉顺序
- 焦点状态清晰可见
- 支持 Enter/Space 触发操作

#### 屏幕阅读器支持

```html
<!-- 使用语义化标签 -->
<nav>...</nav>
<main>...</main>
<aside>...</aside>

<!-- 添加 ARIA 标签 -->
<button aria-label="关闭对话框">×</button>

<!-- 图片添加 alt 文本 -->
<img src="chart.png" alt="2024年销售趋势图" />
```

## 🎯 用户体验检查清单

### 可用性

- [ ] 主要功能 3 次点击内可达
- [ ] 错误信息清晰且提供解决方案
- [ ] 支持撤销操作
- [ ] 加载时间 < 3 秒
- [ ] 表单支持键盘操作

### 一致性

- [ ] 使用统一的设计语言
- [ ] 相同功能使用相同交互
- [ ] 术语和标签保持一致
- [ ] 颜色使用遵循规范

### 反馈

- [ ] 操作有即时反馈
- [ ] 进度有明确指示
- [ ] 成功/失败状态明确
- [ ] 帮助信息易于获取

### 容错

- [ ] 危险操作有确认
- [ ] 输入有格式提示
- [ ] 错误可恢复
- [ ] 数据自动保存
