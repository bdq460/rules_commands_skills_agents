# 校对检查清单

本文档提供产品开发各阶段的校对检查清单，确保每个阶段交付物的质量。

## 阶段1：需求提出 - 校对清单

### 校对人：客户代表 - 需求提出

**自审检查项**：

- [ ] 需求描述清晰，无歧义表达
- [ ] 使用客户语言，避免技术术语
- [ ] 验收标准明确具体、可衡量
- [ ] 业务价值主张清晰
- [ ] 包含完整的业务场景
- [ ] 识别并列出主要约束条件

**质量检查项**：

- [ ] 无拼写错误
- [ ] 语法正确，表达流畅
- [ ] 逻辑连贯，前后一致
- [ ] 格式规范，层次清晰
- [ ] 使用编号和列表增强可读性

### 校对人：需求分析师 - 需求提出

- [ ] 需求结构化、分类清晰
- [ ] 功能需求完整，无遗漏
- [ ] 非功能需求全面
- [ ] 需求优先级明确（P0/P1/P2/P3）
- [ ] 用户故事格式正确
- [ ] 验收标准具体可测试

### 校对人：产品专家 - 需求提出

- [ ] 产品定位准确
- [ ] 功能清单完整
- [ ] 核心价值主张清晰
- [ ] 与原始需求一致
- [ ] 设计方向明确

- [ ] Markdown格式正确

### 内容质量

- [ ] 无拼写错误
- [ ] 无语法错误
- [ ] 逻辑清晰
- [ ] 表达准确
- [ ] 示例完整

### 交付物完整性

- [ ] 所有必需文件已创建
- [ ] 文件格式正确
- [ ] 文件大小合理

### 一致性检查

- [ ] 与前阶段输出一致

- [ ] 编码风格一致
