# 客户代表沟通模板

本文档提供客户代表在需求沟通和业务场景描述时的标准模板。

## 📋 需求描述模板

### 基础需求模板

```markdown
# 需求标题：[简洁描述需求核心]

## 1. 背景说明

- **业务背景**：[描述当前业务现状]
- **提出时间**：[YYYY-MM-DD]
- **优先级**：[高/中/低]
- **期望上线时间**：[YYYY-MM-DD]

## 2. 业务场景

### 场景描述

[详细描述用户在什么情况下会使用这个功能]

### 用户角色

- **主要用户**：[谁会使用这个功能]
- **使用频率**：[每天/每周/每月使用多少次]
- **使用环境**：[PC端/移动端/其他]

## 3. 当前痛点

| 痛点    | 影响程度 | 发生频率       | 影响范围       |
| ------- | -------- | -------------- | -------------- |
| [痛点1] | 高/中/低 | 频繁/偶尔/少见 | [影响多少用户] |
| [痛点2] | 高/中/低 | 频繁/偶尔/少见 | [影响多少用户] |

## 4. 期望方案

### 期望效果

[描述期望达到的效果]

### 成功标准

- [ ] [可衡量的标准1]
- [ ] [可衡量的标准2]
- [ ] [可衡量的标准3]

## 5. 补充说明

[其他需要说明的内容]

```

### 问题反馈模板

```markdown
# 问题反馈：[问题简述]

## 问题描述

- **发现时间**：[YYYY-MM-DD HH:mm]
- **影响范围**：[影响多少用户/业务]
- **紧急程度**：[紧急/重要/一般]

## 复现步骤

1. [步骤1]

2. [步骤2]

3. [步骤3]

## 期望行为

[描述用户期望系统应该如何表现]

## 实际行为

[描述系统实际的表现]

## 截图/录屏

[如有，请附上]

```

## 📝 业务场景描述模板

### 用户故事格式

```markdown
作为 [用户角色]，
我想要 [完成某个目标/功能]，
以便 [获得某种价值/解决某个问题]。

### 验收标准

- [ ] Given [前置条件]，When [执行操作]，Then [预期结果]
- [ ] Given [前置条件]，When [执行操作]，Then [预期结果]

```

### 场景详细描述

```markdown
## 场景：[场景名称]

### 参与者

- **主角**：[主要用户角色]
- **配角**：[其他相关角色]

### 前置条件

- [条件1]
- [条件2]

### 主要流程

1. [用户操作1] → [系统响应1]

2. [用户操作2] → [系统响应2]

3. [用户操作3] → [系统响应3]

### 异常流程

- **异常情况1**：[发生条件] → [处理方式]
- **异常情况2**：[发生条件] → [处理方式]

### 后置条件

- [完成后的状态1]
- [完成后的状态2]

```

## 📊 优先级评估矩阵

### 需求优先级评估

| 评估维度           | 权重 | 评分(1-5) | 加权得分 |
| ------------------ | ---- | --------- | -------- |
| 业务价值           | 30%  |           |          |
| 用户影响面         | 25%  |           |          |
| 紧急程度           | 20%  |           |          |
| 实现复杂度（反向） | 15%  |           |          |
| 战略契合度         | 10%  |           |          |
| **总分**           | 100% |           |          |

### 优先级分类

- **P0（紧急）**：总分 ≥ 4.5，需立即处理
- **P1（高优）**：总分 4.0-4.4，本迭代必须完成
- **P2（中优）**：总分 3.0-3.9，计划内完成
- **P3（低优）**：总分 < 3.0，待排期

## 🤝 沟通话术参考

### 需求澄清

```list
我想确认一下我的理解是否正确：
您希望实现的是 [复述需求]，
主要目的是 [复述目标]，
使用场景是 [复述场景]，
对吗？

```

### 期望管理

```list
关于这个需求，我需要跟您说明：
- 我们可以做到：[可实现的部分]
- 可能需要调整：[需要协商的部分]
- 暂时无法实现：[限制说明]

您看这样的方案是否可以接受？

```

### 变更确认

```list
收到您的变更请求：
- 原需求：[原内容]
- 变更为：[新内容]
- 变更原因：[原因]

这个变更可能会影响：[影响说明]
预计需要额外 [时间] 来完成。

请确认是否继续？

```

## 📌 常用检查清单

### 需求提交前检查

- [ ] 需求背景描述清晰
- [ ] 业务场景有具体示例
- [ ] 痛点有数据支撑（如有）
- [ ] 期望方案可量化
- [ ] 优先级已评估
- [ ] 关联需求已标注
- [ ] 验收标准已定义

### 验收确认检查

- [ ] 功能符合需求描述
- [ ] 用户流程顺畅
- [ ] 异常情况处理合理
- [ ] 性能满足预期
- [ ] 用户体验可接受
